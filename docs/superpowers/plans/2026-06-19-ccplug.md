# ccplug v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A fast Rust CLI (`ccplug list|status|enable|disable`) that manages which globally-installed Claude Code plugins/skills are enabled per project, writing a chosen subset into the settings cascade.

**Architecture:** Read global inventory from `~/.claude/plugins/installed_plugins.json` + each plugin's `skills/*/SKILL.md`. Resolve self-describing targets (`plugin` | `plugin:skill` | `plugin:*`) against that inventory. Mutate the chosen scope's `settings.json` via read-modify-write on a `serde_json::Value` (never wholesale-rewrite; preserve all unrelated keys). Same core path for human and agent — flags differ, logic doesn't.

**Tech Stack:** Rust 2021, clap derive, serde/serde_json, comfy-table, anyhow, dirs, rayon (parallel SKILL.md parse). Tests: assert_cmd + tempfile.

**User decisions (already made):**
- v1 scope only: `list`, `status`, `enable`, `disable`. No extraction/`sync`/`pick`/presets (v2).
- Plugin-owned skill toggle = structured `unsupported-v1` result, never crashes the batch.
- Mirror `csess` conventions (file split, profile, metadata, install.sh/README/CHANGELOG/SKILL/LICENSE).
- Default scope = `project` (`.claude/settings.json`).
- Personal git account, already configured. Do NOT push or create GitHub repo without explicit ask.

## Task 1: CLI surface (clap derive)

**Goal:** Full arg surface so every command parses, even before handlers do real work.

**Files:**
- Modify: `src/cli.rs`

**Acceptance Criteria:**
- [ ] `ccplug list --json`, `ccplug status`, `ccplug enable a b --dry-run`, `ccplug disable --stdin`, `--scope user` all parse
- [ ] `--scope` is an enum {project, local, user}, default project
- [ ] targets are positional `Vec<String>` on enable/disable

**Verify:** `cargo build` + a parse test per subcommand

**Design:**
```rust
#[derive(Parser)]
#[command(name="ccplug", version, about="Manage per-project Claude Code plugins & skills")]
pub struct Cli { #[command(subcommand)] pub command: Command }

#[derive(Subcommand)]
pub enum Command {
    List(CommonFlags),
    Status(CommonFlags),
    Enable(MutateArgs),
    Disable(MutateArgs),
}

#[derive(Args, Clone)]
pub struct CommonFlags {
    #[arg(long)] pub json: bool,
    #[arg(long, value_enum, default_value_t = Scope::Project)] pub scope: Scope,
    #[arg(long, hide = true)] pub home_dir: Option<String>,   // test override of ~/.claude
    #[arg(long, hide = true)] pub project_dir: Option<String>,// test override of cwd
}

#[derive(Args, Clone)]
pub struct MutateArgs {
    pub targets: Vec<String>,
    #[command(flatten)] pub common: CommonFlags,
    #[arg(long)] pub from: Option<String>,
    #[arg(long)] pub stdin: bool,
    #[arg(long)] pub dry_run: bool,
}

#[derive(ValueEnum, Clone, Copy, PartialEq)]
pub enum Scope { Project, Local, User }
```
`home_dir`/`project_dir` hidden overrides keep tests hermetic (csess uses `--projects-dir` the same way).

**Steps:**
- [ ] Write the enums/structs above.
- [ ] Parse test: assert each subcommand variant parses with representative flags.

## Task 3: Target resolution

**Goal:** Turn raw target strings into typed, resolved outcomes against the inventory — the core non-trivial logic.

**Files:**
- Modify: `src/target.rs`
- Test: `src/target.rs` (#[cfg(test)])

**Acceptance Criteria:**
- [ ] `"vercel"` → `Plugin{id:"vercel@..."}`; bare name resolved to full id
- [ ] `"vercel@mkt"` accepted directly
- [ ] Ambiguous bare name (same name in 2 marketplaces) → `Err`/unresolvable result requiring full id
- [ ] Unknown plugin → unresolvable result with clear reason
- [ ] `"vercel:nextjs"` where nextjs is plugin-owned → `Unsupported{reason:"unsupported-v1"}`
- [ ] `"x:skillname"` where skillname matches a loose skill → `LooseSkill{name}`
- [ ] `"vercel:*"` → `Plugin` + glob note flag

**Verify:** `cargo test target` — table of inputs → expected `Resolved` against a fixture inventory.

**Design:**
```rust
pub enum Kind { Plugin, PluginGlob, PluginSkill, LooseSkill }
pub struct Resolved {
    pub raw: String,
    pub kind: Kind,
    pub plugin_id: Option<String>,  // for Plugin/Glob/PluginSkill
    pub skill: Option<String>,      // for *Skill
    pub ok: bool,                   // false = unresolvable or unsupported-v1
    pub reason: Option<String>,     // e.g. "unsupported-v1", "unknown-plugin", "ambiguous"
}

pub fn resolve(raw: &str, plugins: &[Plugin], loose: &[Skill]) -> Resolved;
```
Logic:
1. Split on first `:` → (left, right?).
2. Resolve `left` to a plugin: if contains `@`, match id exactly; else match by `name` — 0 → unknown, 1 → that id, >1 → ambiguous.
3. No `:` → `Plugin`.
4. `:*` → `PluginGlob` (carries plugin_id; ok=true; note appended at output).
5. `:skill`:
   - if plugin resolved AND skill in that plugin's skills → `PluginSkill`, ok=false, reason `unsupported-v1`.
   - else if skill name in `loose` → `LooseSkill`, ok=true.
   - else ok=false, reason `unknown-skill`.

**Steps:**
- [ ] Implement `resolve` per logic above.
- [ ] Tests: each AC row with a small in-memory `Vec<Plugin>`/`Vec<Skill>` fixture.

## Task 4: Settings read / merge / write

**Goal:** Read the cascade for `status`; do safe read-modify-write of one scope file for mutations, preserving all unrelated keys.

**Files:**
- Modify: `src/settings.rs`
- Test: `src/settings.rs` (#[cfg(test)])

**Acceptance Criteria:**
- [ ] `scope_path(scope, project, home)`: project→`<project>/.claude/settings.json`, local→`<project>/.claude/settings.local.json`, user→`<home>/.claude/settings.json`
- [ ] `read_value(path)` → `serde_json::Value::Object` (missing file → `{}`)
- [ ] `apply` sets only `enabledPlugins[id]` / `skillOverrides[name]`; a file with other keys (`$schema`, `permissions`, …) keeps every one of them
- [ ] `.bak` written once before first real mutation (skipped on dry-run)
- [ ] merged cascade: user < project < local, per-key override; tracks source scope per key
- [ ] dry-run returns the would-be diff string, writes nothing

**Verify:** `cargo test settings` — round-trip: seed a file with extra keys, apply a plugin toggle, assert extra keys intact + value changed + `.bak` exists.

**Design:**
```rust
pub fn scope_path(scope: Scope, project: &Path, home: &Path) -> PathBuf;

pub struct Effective { pub plugins: BTreeMap<String,(bool,Scope)>, pub overrides: BTreeMap<String,String> }
pub fn effective(project: &Path, home: &Path) -> Result<Effective>; // merge 3 files

// mutation
pub struct Change { pub set_plugin: Vec<(String,bool)>, pub set_override: Vec<(String,String)> }
pub fn apply(path: &Path, change: &Change, dry_run: bool) -> Result<Applied>;
pub struct Applied { pub wrote: bool, pub diff: String, pub backup: Option<PathBuf> }
```
- `read_value`: `fs::read` → `serde_json::from_str`; ENOENT → `json!({})`.
- `apply`: clone value, ensure `enabledPlugins`/`skillOverrides` are objects, insert keys, `to_string_pretty`. If changed & !dry_run: write `.bak` (copy original if exists) then write new. `// ponytail: serde_json reformats (drops comments/exact spacing) but preserves all keys — acceptable; upgrade to a format-preserving editor only if a user keeps JSONC comments`.
- diff: cheap line diff old vs new pretty (or just show new `enabledPlugins`/`skillOverrides` block). Keep minimal.

**Steps:**
- [ ] `scope_path`, `read_value`, `effective` (merge order user→project→local, record source).
- [ ] `apply` with backup + dry-run.
- [ ] Tests: preserve-keys round-trip, missing-file create, dry-run no-write, cascade override+source.

## Task 5: Command handlers + wiring

**Goal:** Implement `list`, `status`, `enable`, `disable` end-to-end, including target input (`--from`/`--stdin`/positional), partial-batch semantics, warnings, exit codes.

**Files:**
- Modify: `src/main.rs`, `src/output.rs`

**Acceptance Criteria:**
- [ ] `list` shows every plugin + enabled state + skills (table + `--json` per §6)
- [ ] `status` shows effective plugins + source + skillOverrides for cwd (§6)
- [ ] `enable`/`disable` resolve targets, build `Change`, call `apply`, print `{file,scope,dryRun,results,warnings}` (§6)
- [ ] targets come from positional OR `--from FILE` (JSON array) OR `--stdin` (JSON array)
- [ ] `PluginSkill` target → result `ok:false reason:"unsupported-v1"`, batch continues
- [ ] `PluginGlob` → toggles whole plugin + warning note "per-skill plugin control unavailable in v1"
- [ ] disabling a plugin that provides mcp/lsp/agents → warning listing them
- [ ] exit non-zero only if EVERY target failed (else 0); always exit non-zero on hard errors (bad file, bad JSON input)

**Verify:** `cargo test` (integration in `tests/cli.rs`) — enable a plugin into a tempdir project, assert settings written + JSON result shape; mixed batch (one plugin + one plugin-skill) exits 0 with one ok + one unsupported.

**Design / flow (enable/disable):**
```
targets = gather(positional, from, stdin)        // error if none
plugins = inventory::load(home); loose = loose_skills(project, home)
resolved = targets.map(|t| target::resolve(t, &plugins, &loose))
change = Change from resolved where ok && (Plugin|PluginGlob|LooseSkill)
warnings = for each disabled plugin with mcp/lsp/agents provides; glob note
applied = settings::apply(scope_path, &change, dry_run)
output = render results (§6); exit_code = if all !ok {1} else {0}
```
`enable` vs `disable` differ only by the boolean / `"on"|"off"`.

**Steps:**
- [ ] `gather_targets`: positional non-empty → use; else `--from` read file; else `--stdin` read stdin; parse JSON array → `Vec<String>`. Error if still empty.
- [ ] `cmd_list`, `cmd_status`, `cmd_mutate(enabled: bool)`.
- [ ] `output.rs`: `list_table`, `list_json`, `status_table`, `status_json`, `mutate_json`, `mutate_human`.
- [ ] Integration tests in `tests/cli.rs` using `--home-dir`/`--project-dir` overrides + tempdir.
